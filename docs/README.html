Task Manager - Projeto de Estudo React

Sobre o Projeto

Este é um projeto de estudo focado em boas práticas de organização e arquitetura em React. O projeto implementa um gerenciador de tarefas que serve como exemplo prático de várias boas práticas e padrões de projeto modernos.

Boas Práticas Implementadas

1. Componentes "Burros" (Presentational Components)

Separação clara entre componentes de apresentação e lógica:

  • TaskCard: Apenas renderiza dados, sem lógica de negócio
  • StatCard: Componente reutilizável para exibição de estatísticas
  • Lógica de negócio movida para hooks e contextos

2. Custom Hooks

Encapsulamento de lógica reutilizável:

  • useTaskStats: Cálculos de estatísticas das tarefas
  • useTaskFilters: Gerenciamento de filtros com URL
  • useTasks: Gerenciamento de estado com React Query

3. Organização por Feature

Estrutura de pastas organizada por funcionalidade:

  • components/tasks/: Componentes específicos de tarefas
  • components/dashboard/: Componentes do dashboard
  • components/ui/: Componentes base reutilizáveis

4. Gerenciamento de Estado

Múltiplas estratégias de estado:

  • React Query para estado do servidor
  • Context API para estado global
  • URL para estado de filtros
  • Estado local onde apropriado

5. TypeScript e Tipagem

Uso extensivo de TypeScript:

  • Interfaces bem definidas para dados
  • Types para valores enumerados
  • Props tipadas em componentes
  • Tipagem de hooks e contextos

6. Mock de API

Desenvolvimento independente do backend:

  • MSW para interceptação de requisições
  • Dados mock realistas
  • Simulação de latência e erros

7. Componentização Eficiente

Decomposição inteligente de componentes:

  • Componentes pequenos e focados
  • Reutilização através de composição
  • Props para configuração
  • Children para flexibilidade

8. Estilização Moderna

Abordagem moderna para estilos:

  • Tailwind CSS para utilidades
  • Variáveis CSS para temas
  • Componentes shadcn/ui como base
  • Sistema de cores consistente

9. Feedback e UX

Atenção à experiência do usuário:

  • Estados de loading
  • Tratamento de erros
  • Confirmações de ações
  • Feedback visual

10. Código Limpo

Práticas de código limpo:

  • Nomes descritivos
  • Funções pequenas e focadas
  • DRY (Don't Repeat Yourself)
  • Separação de responsabilidades

Padrões de Projeto

  • Provider Pattern: Usado no ThemeProvider e TaskProvider
  • Compound Components: Implementado nos componentes de formulário
  • Render Props: Usado em componentes de lista
  • Custom Hooks: Para reutilização de lógica
  • Container/Presentational: Separação de lógica e apresentação

Conclusão

Este projeto demonstra como diferentes padrões e práticas podem ser combinados para criar uma aplicação React moderna, manutenível e escalável. Cada decisão de arquitetura foi tomada considerando o equilíbrio entre complexidade e benefício, resultando em um código organizado e de fácil manutenção.
